## Local Preferences

- Use valid RED for behavior changes, a fresh passing baseline for
  behavior-preserving work, and proportionate alternate evidence otherwise.
- The evidence choice is semantic and needs no named record.
- Preserve user and prior-stage work instead of reverting it to manufacture RED.
- Correct obvious test or command mistakes in place; route repeated unexplained
  failures to evidence debugging.
- Prefer one focused check and the cheapest directly coupled evidence over the
  whole suite during every cycle.
- Report before, after, and remaining evidence gaps.
