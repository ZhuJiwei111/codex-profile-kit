# Personal Writing Polish

Improve the expression of locked text without taking ownership of its facts,
reasoning, evidence, decisions, or required structure.

This is an expression-only workflow. A request that needs new claims, changed
reasoning, or structural redesign returns to the content owner first.

## Entry

Use this skill when both are true:

- the text is supplied, identified, or already settled by its content owner; and
- the user clearly asks for a language transformation such as polish, rewrite,
  tighten, humanize, or voice matching.

Natural-language intent is enough; the user need not name the skill. Do not
trigger merely because an ordinary answer could sound better.

If a substantive claim, evidence boundary, decision, or required structure is
still unresolved, return to the content-owning workflow before polishing.

## Lock The Meaning

Before rewriting, preserve:

- facts, claims, stance, and the distinction between observation, inference,
  hypothesis, and recommendation;
- evidence and causal strength;
- scope, conditions, exceptions, negations, limitations, non-goals, uncertainty,
  and unknowns;
- numbers, units, dates, quotations, citations, links, and attribution;
- code, commands, paths, APIs, identifiers, LaTeX, tables, and domain terms; and
- actions, obligations, deadlines, venue rules, templates, and format limits.

Do not invent support, explanations, examples, opinions, personality, or next
steps. Do not remove a necessary caveat or turn a negative claim into an
attractive positive claim unless they are logically equivalent and equally
supported.

## Rewrite

Follow the requested voice, audience, artifact, language, and format. Without
more specific instructions:

- preserve the source language;
- improve clarity, referents, sentence flow, and rhythm;
- remove true repetition and mechanical or defensive phrasing only when meaning
  stays unchanged;
- preserve the order and attachment of material propositions; and
- treat punctuation, passives, headings, lists, and hedges as contextual choices,
  not automatic faults.

Use a supplied voice sample only as an expression reference. Do not infer its
author's biography, beliefs, or experiences.

## Check And Return

Compare the rewrite with the source:

- every material claim retains the same truth conditions;
- evidence strength, causal force, scope, uncertainty, and negation remain
  intact;
- protected literals and format constraints remain unchanged; and
- smoother wording introduced no unsupported content.

Return only the rewritten text by default. Give an audit, alternatives, or a
change rationale only when requested or when a semantic conflict must be
surfaced. Edit a file only when that file change is authorized, and touch only
the requested prose surface.

Read references/source-notes.md only when maintaining provenance or revising
this skill.
